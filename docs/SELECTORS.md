# Селекторы в Scenaria

Scenaria использует движок Playwright. В шагах селектор — строка в кавычках: `нажимаю "#login"`, `вижу "text=Сохранить"`.

## Приоритет при записи

При live-записи порядок стратегий (см. `internal/selector/strategies.go`):

| Стратегия | Пример | Когда использовать |
|-----------|--------|-------------------|
| **testid** | `[data-testid="submit"]` | Стабильный атрибут в вашем приложении |
| **id** | `#login-form` | Уникальный `id` на странице |
| **aria / role** | `role=button[name="Войти"]` | Доступность, кнопки и ссылки |
| **label** | `label=Email` | Поля ввода с `<label>` |
| **placeholder** | `[placeholder="Поиск…"]` | Поля без label |
| **text** | `text="Каталог"` | Видимый текст (лучше, чем `:has-text` в CSS) |

Для кликов: `testid → id → aria → title → contextual → text`.  
Для полей: `testid → id → label → placeholder → aria → name`.

## Цепочки (`>>`)

Если элемент внутри блока, используйте цепочку Playwright:

```gherkin
Когда нажимаю "nav >> text=Товары"
Тогда вижу "main >> h1"
```

Scenaria разрешает цепочку через `ResolveChainedLocator`: сначала контейнер, затем цель.

## Рекомендации для автотестов

1. **Добавляйте `data-testid`** на кнопки, формы и ключевые блоки — это самый устойчивый вариант.
2. **Предпочитайте role + name** вместо длинных CSS: `role=link[name="Документация"]`.
3. **Избегайте** селекторов только по тегу (`div`, `img`) и длинных путей из записи — укорачивайте вручную после записи.
4. **Не полагайтесь на UI Scenaria** при записи на своём сайте — панель записи помечена `data-scenaria-ui` и фильтруется.
5. **Меню по hover**: если клик не срабатывает, шаг `навожу` перед кликом или селектор с `>>` на пункт меню.

## Переменные `{{имя}}`

В селекторах и значениях можно подставлять переменные сценария, запомненные поля и `{{env:VAR}}`:

```gherkin
Когда нажимаю "{{button_id}}"
И заполняю "{{email}}" в "input[type=email]"
```

Ограничения:

- Вложенность разрешается до 16 проходов; циклические ссылки (`a → b → a`) дают ошибку.
- Незакрытый `{{` без `}}` остаётся в строке как есть.
- Сложные вложенные шаблоны лучше разбивать на шаги `запоминаю … как "имя"`.

## Проверка селекторов

В IDE: **Селекторы на странице** (валидация открытого проекта в браузере) или **Проверка в браузере** из меню.

Шаги `проверяю url` / `url содержит` ждут совпадения до ~5 с (с учётом дедлайна контекста) — удобно после SPA-навигации. Повтор шага из retry-политики добавляет ещё попытки.

CLI:

```bash
scenaria validate ./examples --browser
```

См. также `docs/RECORDER.md` — как записываются селекторы и что нормализуется после записи.
